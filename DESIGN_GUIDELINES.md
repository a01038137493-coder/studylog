# 로그 디자인 지침서 (Design Guidelines)

재수학원 전용 **학생 목표·성과 추적 웹앱** 로그의 디자인 시스템 문서입니다.
출처는 실제 코드(`css/style.css`)이며, 새 화면·컴포넌트를 만들 때 이 문서를 기준으로 합니다.

> 핵심 한 줄: **“출석이 아니라, 목표 대비 성과를 관리한다.”**
> 데이터를 담백하고 또렷하게 보여주는 것이 디자인의 목적입니다.

---

## 1. 디자인 원칙

1. **모노크롬 우선** — 기본은 검정·흰색·회색. 색(주황/초록)은 *의미*를 전달할 때만 쓴다(경고·성공). 장식용 색 금지.
2. **데이터가 주인공** — 숫자·달성률·상태가 가장 먼저 읽혀야 한다. 위계를 분명히.
3. **담백한 미니멀** — 여백을 넉넉히. 테두리보다 여백과 그림자로 구분.
4. **태블릿·모바일 우선** — 학원 환경. 터치 영역을 크게(주요 버튼 최소 높이 58px).
5. **다크모드 없음** — 밝은 단일 테마. (단, 학생 홈은 ‘뉴모피즘’ 부드러운 테마를 별도 사용 → 6장 참고)
6. **격려하는 톤** — 압박보다 “오늘 한 걸음”을 인정. (10장 보이스 참고)

---

## 2. 로고 / 워드마크

- 워드마크: **로그** (한글), 영문 시스템명 *Goal Tracker*.
- 상단바 표기: `font-weight:700`, `letter-spacing:-0.02em`, 약 1.05rem.
- 광고/배지용 심볼: `● 로그` (앞에 강조색 점). 점 색은 코랄(#ff5a4d) 또는 검정.
- 로고 파일: `assets/logo.svg` (배경 투명). 최소 여백 = 로고 높이의 50%.

## 3. 컬러 시스템

### 3.1 브랜드 / 뉴트럴 (기본 UI)
| 토큰 | HEX | 용도 |
|---|---|---|
| `--primary` / `--navy` | `#111111` | 주요 버튼·강조 텍스트·아이콘 |
| `--primary-600` | `#000000` | 가장 진한 강조(hover 등) |
| `--primary-100` | `#ececec` | 연한 강조 배경 |
| `--text` | `#141414` | 본문 텍스트 |
| `--muted` | `#6b6b6b` | 보조 텍스트 |
| `--muted-soft` | `#9a9a9a` | 더 흐린 텍스트·플레이스홀더 |
| `--bg` | `#f4f4f5` | 앱 배경 |
| `--bg-soft` | `#fafafa` | 살짝 들어간 영역 배경 |
| `--card` | `#ffffff` | 카드·표면 |
| `--line` | `#ededed` | 구분선(약) |
| `--line-strong` | `#d6d6d6` | 구분선(강)·점선 테두리 |

### 3.2 상태(의미) 컬러 — *의미 전달에만 사용*
| 토큰 | HEX | 용도 |
|---|---|---|
| `--green` | `#2aa56e` | 성공·완료·달성 |
| `--green-strong` | `#1f8d5d` | 성공 강조 |
| `--green-bg` | `#edf6f1` | 성공 배경 칩 |
| `--orange` | `#ef8e3c` | 경고·주의·미완료 |
| `--orange-strong` | `#d96f1d` | 강한 경고·에러 텍스트 |
| `--orange-bg` | `#fbf1e7` | 경고 배경 칩 |
| `--blue-bg` | `#f0f0f0` | 중립 정보 배경 |

> 규칙: 빨강(에러)은 `--orange-strong`로 대체해 톤을 유지한다. 파랑·보라 등 임의 색 추가 금지.

### 3.3 학생 대시보드 ‘뉴모피즘’ 테마 (`body.nm`)
학생 홈은 부드러운 블루그레이 + 골드 포인트의 별도 테마를 쓴다.
| 역할 | HEX |
|---|---|
| 표면/배경 | `#d6dde9` |
| 본문 텍스트 | `#2c3340` / `#3a4252` |
| 흐린 텍스트 | `#8893a5` |
| 보조 강조 | `#5a6b86` |
| 골드 포인트(진행·강조) | `#c2862a` (보조 `#9a6a1e`, `#e3a23e`) |

## 4. 타이포그래피

- **서체**: `Pretendard Variable, Pretendard` → 폴백 `-apple-system, "Apple SD Gothic Neo", "Malgun Gothic", "Segoe UI", Roboto, sans-serif`
- **기준 크기**: `html { font-size:14px }` → 모든 `rem`은 14px 기준(전체 UI를 의도적으로 압축).
- **기본 본문**: `line-height:1.55`, `letter-spacing:-0.01em`
- **제목**: `h1,h2,h3` `letter-spacing:-0.02em`

### 타입 스케일 (14px 기준)
| 역할 | 크기 | 굵기 | 비고 |
|---|---|---|---|
| 페이지 타이틀 | `1.55rem` (~21.7px) | 700~800 | 화면 상단 대표 제목 |
| 섹션/카드 제목 | `1.05rem` (~14.7px) | 700 | `--navy` 색 |
| 본문 | `1rem` (14px) | 400~500 | |
| 보조 텍스트 | `0.9–0.92rem` | 400~500 | `--muted` |
| 마이크로(라벨/배지) | `0.74–0.82rem` | 700~800 | 대문자 느낌 강조 |
| 광고/임팩트 헤드라인 | 별도(Black Han Sans 등) | 900 | 앱 내부 아님, 마케팅 전용 |

- 굵기 사용: 400(본문) · 500(약강조) · 600(버튼) · 700(제목) · 800~900(수치·라벨 강조).
- 숫자(달성률·시간·D-day)는 `font-variant-numeric: tabular-nums` 권장(정렬 안정).

## 5. 레이아웃 · 간격 · 모서리

### 5.1 모서리 반경(Radius)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--radius` | `20px` | 큰 카드·시트 |
| `--radius-sm` | `12px` | 버튼·작은 카드·입력 |
| `--radius-xs` | `9px` | 칩·작은 컨트롤 |

### 5.2 컨테이너 폭
| 클래스 | max-width | 용도 |
|---|---|---|
| `.container` | 700px | 기본 폼/문서 화면 |
| `.container--wide` | 1140px | 넓은 표·대시보드 |
| `.home-layout` | 980px | 학생 홈(2열) |
| `.home-wide` | 1220px | 학생 홈 위젯 캔버스 |

- 모두 `margin:0 auto`로 가운데 정렬.

### 5.3 간격 스케일
주로 쓰는 값: **4 · 6 · 8 · 12 · 16 · 22 px**. (gap·padding 모두 이 단위 안에서.)

### 5.4 반응형 브레이크포인트
- `min-width: 768px` — 태블릿/PC(2열·넓은 레이아웃 전환)
- `max-width: 480px` — 소형 모바일 보정
- 모바일 우선으로 작성하고 768px에서 확장.

### 5.5 위젯 격자(학생 홈)
- 칸 크기: **118 × 122 px**, 간격 **12px**, 가운데 정렬.
- 위젯 배치는 화면 ‘열 수’별로 따로 저장(기기 전환 시 흐트러짐 방지).

## 6. 그림자 (Elevation)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--shadow-xs` | `0 1px 2px rgba(0,0,0,.05)` | 칩·작은 표면 |
| `--shadow` | `0 4px 16px rgba(0,0,0,.07)` | 기본 카드 |
| `--shadow-md` | `0 8px 24px rgba(0,0,0,.09)` | 떠 있는 카드·메뉴 |
| `--shadow-lg` | `0 16px 40px rgba(0,0,0,.16)` | 모달·바텀시트 |
| `--ring` | `0 0 0 3px rgba(0,0,0,.14)` | 포커스 링 |

> 경계는 **선보다 그림자**로. 진한 테두리 남발 금지.

## 7. 컴포넌트

### 7.1 버튼
- 기본 `.btn`: `padding:12px 18px`, `border-radius:12px`(sm), `font-weight:600`, `font-size:1rem`. 누르면 살짝 가라앉음(`translateY(1px) scale(.995)`).
- 변형:
  - `.btn--primary` — 검정 배경·흰 글자. **주요 행동 1개**에만.
  - `.btn--ghost` — 어두운 바 위 반투명 흰색(상단바 등).
  - `.btn--ghost-dark` — 흰 배경 + `--line-strong` 테두리(밝은 화면의 보조 버튼).
  - `.btn--outline` — 흰 배경 + 점선 테두리(‘추가/선택’ 등 보조).
  - 크기: `.btn--lg`(padding 16×22, **min-height 58px**, radius 14 — 모바일 주 행동), `.btn--sm`(padding 8×13, radius 9), `.btn--icon`(정사각 아이콘), `.btn--block`(가로 100%).
- 한 화면에 `--primary` 버튼은 **하나**. 나머지는 ghost/outline.

### 7.2 카드 (`.card`)
- 배경 `--card`(#fff), `--radius`(20), `--shadow`. 내부 padding 16~20px.
- 카드 제목 `.card__title`: 1.05rem/700/`--navy`, 아래 여백 16px.

### 7.3 입력 필드 (`.field`)
- 라벨(`.field__label`) + 인풋(`.field__input`). 인풋 `border-radius` sm(12), 큰 터치 높이.
- 포커스 시 `--ring`. 에러 메시지 `.form__error`는 `--orange-strong`.

### 7.4 칩 / 배지
- 작은 라운드(999px 또는 xs). 상태색 배경(`--green-bg`/`--orange-bg`) + 진한 텍스트.
- 텍스트 0.74~0.82rem / 700~800.

### 7.5 상단바 (`.topbar`)
- 어두운 배경 + 워드마크(좌) + 액션(우, `.btn--ghost`/`.btn--sm`).

### 7.6 토스트 / 모달 / 바텀시트
- 토스트: 화면 하단 중앙, 어두운 배경, 그림자 md.
- 모달/바텀시트: 오버레이 `rgba(0,0,0,.45)`, 시트 `--radius`(상단 20), `--shadow-lg`, `max-height:80vh` 스크롤.

### 7.7 진행/달성 바
- 트랙 `#e3e8f1`(또는 nm 테마 반투명), 채움은 강조색. 높이 6~12px, radius 999.

---

## 8. 아이콘 · 이모지

- 라인 아이콘(SVG, `stroke-width≈2.2`, 둥근 끝) 위주. `assets/icon/` 사용.
- 이모지는 위젯·카드의 *친근한 라벨*에 한해 허용(✅ 오늘 과제 / ⏱️ 공부시간 / 📈 달성률 / 📅 달력).
- 본문·버튼에는 이모지 남용 금지.

---

## 9. 모션

- 짧고 절제: 트랜지션 `0.08~0.18s ease`. 버튼 누름·hover·등장 정도.
- `prefers-reduced-motion` 존중(애니메이션 축소 분기 제공).
- 화려한 연출 대신 ‘반응이 빠르다’는 느낌에 집중.

---

## 10. 보이스 & 톤 (UX 라이팅)

- **언어**: 한국어, 존댓말. 간결하고 따뜻하게.
- **태도**: 압박·평가 대신 *격려와 사실*. “미달”보다 “오늘 65% 달성 — 내일은 오전 집중 추천”.
- **수치는 솔직하게**, 메시지는 **다음 행동 하나**를 제시.
- 용어 통일: **체크인**(오늘 플랜) / **체크아웃**(오늘 성과) / **달성률** / **주간 목표** / **주간 회고** / **상담 필요**.
- 빈 상태 문구 예: “오늘 플랜이 없어요 — 체크인하고 시작해요.”
- 버튼 라벨은 동사+목적: “오늘 플랜 저장”, “성과 기록하기”, “체험하기 →”.

---

## 11. 접근성

- 본문 대비 AA 이상(검정 텍스트/흰 배경 기본 충족). 흐린 텍스트(`--muted-soft`)는 *보조 정보에만*.
- 터치 타깃 최소 44×44px(주 버튼 58px).
- 모든 인터랙티브 요소에 포커스 표시(`--ring`)와 `aria-label`(아이콘 버튼 필수).
- 색만으로 의미 전달 금지 — 상태색에는 항상 텍스트/아이콘 동반.

---

## 12. Do / Don’t

**Do**
- 모노크롬으로 정리하고, 색은 상태에만.
- 큰 터치 영역과 또렷한 위계.
- 그림자로 깊이, 여백으로 구분.
- 숫자는 tabular-nums로 정렬.

**Don’t**
- 임의의 브랜드 색(파랑/보라 등) 추가.
- 한 화면 다중 `--primary` 버튼.
- 진한 테두리 남발 / 정보 과밀.
- 이모지·애니메이션 과용.

```css
/* 뉴트럴 */
--primary:#111111; --primary-600:#000000; --primary-100:#ececec;
--text:#141414; --muted:#6b6b6b; --muted-soft:#9a9a9a;
--bg:#f4f4f5; --bg-soft:#fafafa; --card:#ffffff;
--line:#ededed; --line-strong:#d6d6d6;
/* 상태 */
--green:#2aa56e; --green-strong:#1f8d5d; --green-bg:#edf6f1;
--orange:#ef8e3c; --orange-strong:#d96f1d; --orange-bg:#fbf1e7;
/* 라운드/그림자 */
--radius:20px; --radius-sm:12px; --radius-xs:9px;
--shadow-xs:0 1px 2px rgba(0,0,0,.05); --shadow:0 4px 16px rgba(0,0,0,.07);
--shadow-md:0 8px 24px rgba(0,0,0,.09); --shadow-lg:0 16px 40px rgba(0,0,0,.16);
--ring:0 0 0 3px rgba(0,0,0,.14);
/* 폰트 */
font-family:"Pretendard Variable",Pretendard,-apple-system,"Apple SD Gothic Neo","Malgun Gothic","Segoe UI",Roboto,sans-serif;
/* 학생 홈 뉴모피즘 */
nm-surface:#d6dde9; nm-text:#2c3340; nm-muted:#8893a5; nm-accent:#c2862a;
```
